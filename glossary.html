Glossary

This glossary explains key HTML terms used in my website and in my learning journey:

HTML
HyperText Markup Language — the standard language for creating web pages.
CSS
Cascading Style Sheets — used to style and layout HTML elements.
<img>
Tag used to display images on a webpage. Requires a src and alt attribute.
src
Specifies the path to the image file to be displayed.
alt
Provides alternative text for an image, useful for accessibility and when the image can't load.
<a>
Anchor tag used to create hyperlinks to other pages or websites.
href
Specifies the destination URL of a link.
<em>
Emphasizes text, usually displayed in italics.
<ul>
Defines an unordered list (bulleted list).
<li>
Defines a list item inside a list (ordered or unordered).
HTML Form
A structured set of input fields used to collect user data on a webpage. Common controls include text boxes, dropdowns, radio buttons, and textareas. Forms are essential for user interaction and data submission.
Backend
The part of a website that processes and stores data behind the scenes. In this project, the backend is handled by Formspree, which receives and manages form submissions without needing server-side code.
Formspree
A free service that connects HTML forms to a backend endpoint. It allows developers to collect form responses securely and redirect users to a custom thank-you page after submission.
Action & Method
Attributes used in the <form> tag. Action defines the destination URL where the form data will be sent. Method specifies how the data is sent—usually POST for secure submissions.
Form Submission
The process of sending user-entered data from a form to a backend service.
